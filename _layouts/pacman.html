Core Idea

Dijkstra repeatedly chooses the unvisited tile with the smallest accumulated cost. Once chosen, that tile's best cost is finalized.

Update Rule

new cost = current cost + next tile cost If the new cost is smaller than the old record, update the distance and remember where it came from.

What To Notice

1
Start has distance 0. All other walkable tiles begin as unknown.
2
The purple frontier contains candidate tiles we may visit next.
3
The yellow tile is the cheapest known tile. Its neighbors may get better costs.
4
Green arrows show the returned path from ghost to Pac-Man.

Game Code Link

find_path(cost_map, ghost_pos, pacman_pos) hides this search behind a simple API. The returned list starts at path[0], moves next to path[1], and ends at Pac-Man.
